---
name: research-approach
description: Research modern open-source packages that solve a similar task, analyze their implementation, and recommend an approach. Use when the user asks how to implement or design something and wants real-world examples.
---

1. Inspect the user's project to identify the ecosystem.
2. Find 3–5 current, actively maintained open-source packages from the same ecosystem that solve the user's problem.
3. Present the candidates as a list, each with a link and a brief summary of their approach. Ask the user to select one (or more) packages for a deeper dive.
4. Analyze the selected packages' approaches relentlessly. Walk down each branch of the design tree, resolving dependencies between decisions one by one.
5. Briefly present the packages' implementation approach, with links to specific files and resources. Then, provide your recommended approach to the user's project.
